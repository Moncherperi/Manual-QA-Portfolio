# Тестирование требований по [документации](https://u.netology.ru/backend/uploads/lms/content_assets/file/996/Пользовательский_сценарий_по_процессу_восстановления_пароля_на_сайте_henderson.docx)

## Тестирование требований:
1. В документации указано: Восстановление пароля личного кабинета на сайте возможно через авторизацию в личном кабинете https://henderson.ru/hlogin/, но авторизация - это процесс входа зарегистрированного пользователя, получается несостыковка в формулировке.  
2. Если пользователь начал вводить номер телефона (или e-mail) и только после ввода понял, что пароль не помнит, при нажатии на кнопку Забыли пароль будет ли предложено восстановить через тот способ связи, который уже был введен?  
3. Пользователь вводит номер телефона (8-888-888-88-88), нажимает кнопку “Отправить”. Будет ли использоваться маска для ввода телефона? Предусмотрена ли ошибка при вводе невалидного номера телефона или телефона, которого нет в системе?   
4. Система отображает поле “Проверочный код”. - поле появится ниже поля ввода телефона, или может в отдельном поп-ап окне?  
5. Система отправляет в смс или в вайбер код. Если указано “или” по какому принципу система будет определять, куда направлен код и как пользователь должен это понять? Будет ли отображаться таймер с обратным отсчетом до момента, когда пользователь сможет запросить новый код?  
6. Пользователь вводит 6-ти значный код. Будет использоваться маска ввода кода?  
7. На данной странице пользователь устанавливает новый пароль. Каковы требования к установке пароля? Будет ли достаточно циферного пароля, или требуется более сложная вариация букв, цифр, символов, разного регистра?  
8. Для повторной отправки кода пользователю необходимо нажать кнопку “Код не пришел, повторить отправку”. Кнопка будет сразу видна на экране, или будет появляться только после истечения таймера?  
9. Пользователь нажимает радиокнопку “по email”, система отображает поле для ввода email. Поле отображается в следующем окне или появляется ниже?  
10. Пользователь вводит email, система проводит валидацию, пользователь нажимает кнопку “Отправить”. Предусмотрена  ли ошибка при вводе невалидного e-mail или e-mail, которого нет в системе?  
11. Система отображает сообщение: “На Ваш адрес мы направили письмо со ссылкой. Перейдите по ней, чтобы подтвердить корректность адреса.” Сообщение появляется под полем ввода e-mail, под кнопкой Отправить или в отдельном окне?  
12. Через несколько секунд система отображает кнопку “Письмо не пришло, повторить отправку”. Где будет отображаться кнопка? Она будет появляться, или будет с самого начала в форме ввода e-mail, но будет неактивной?
